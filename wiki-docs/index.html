Wikipediaガチャ

ランダムWikipedia記事カードアプリの制作記録

やったこと

Step 1

Wikipedia APIの調査と検証

REST API（/page/random/summary）とAction API（links取得）の2つを使用。まずcurlで各エンドポイントの挙動を確認し、レスポンス構造を把握してから実装に入った。

ここで学ぶこと: REST API Action API CORS(origin=*) User-Agent
Step 2

カード風UIの実装

トレーディングカード風のレイアウト（300×420px、金枠）で記事を表示。サムネイル画像・タイトル・抜粋テキストをカード内に配置し、視覚的なインパクトを重視した。

ここで学ぶこと: アスペクト比 CSSボーダー カードデザイン
Step 3

リンクワードによる記事遷移

記事内リンクからランダムに3〜5個を選び、クリックで次の記事へ遷移する仕組み。Action APIでリンク一覧を取得し、名前空間フィルタで記事ページのみに絞り込む。

ここで学ぶこと: 記事内リンク抽出 名前空間フィルタ ランダム選択
Step 4

履歴管理とアニメーション

カード切り替え時のフリップアニメーションと、閲覧した記事の履歴スタックを実装。「戻る」ボタンで前のカードに戻れるようにした。

ここで学ぶこと: CSS 3Dフリップ スライドアニメーション 履歴スタック
