論文速報Bot

Nature最新論文をDiscordに自動投稿するBotの制作記録

概要

Nature誌の最新論文を定期的にチェックし、Blueが内容を読んでKaiの興味に合いそうな論文を選び、Discordの #論文速報 チャンネルに日本語要約付きで自動投稿するBot。

制作ステップ

Step 1

RSSフィードの取得

多くの学術ジャーナルはRSSフィードで新着論文を配信している。PythonでこのRSSフィードをパースし、新着論文のタイトル・DOI・著者情報を取得する。

  • feedparserライブラリを使ってRSSフィード（XML形式）を解析
  • フィードURLを指定するだけで、構造化されたデータとして論文情報を取得できる
import feedparser

feed_url = "https://www.nature.com/nature.rss"
feed = feedparser.parse(feed_url)

for entry in feed.entries:
    title = entry.title
    doi = entry.id
    authors = entry.get("author", "Unknown")
ここで学ぶこと: RSSフィードとは何か、XMLデータの構造、Pythonでの外部ライブラリ利用
Step 2

論文のフィルタリング

Natureには幅広い分野の論文が掲載されるため、全論文を投稿するのではなく、BlueがKaiの興味・関心を推定して関連性の高いものを選ぶ。

  • Blueが過去の会話や専門分野から興味の傾向を把握し、フィルタリング基準を動的に調整
  • 投稿済みDOIをJSONファイルで管理し、重複投稿を防止
import json

# 投稿済みDOIの読み込み
with open("posted_dois.json", "r") as f:
    posted = json.load(f)

# 新着のみ抽出（Blueが内容を読んで選別）
new_papers = [
    e for e in feed.entries
    if e.id not in posted
]
ここで学ぶこと: テキストフィルタリング、JSONファイルによる状態管理
Step 3

アブストラクトの取得

RSSフィードにはアブストラクトが含まれない場合がある。そこで出版社のWebページに直接アクセスしてアブストラクトを取得する。

  • 最初はHTTPリクエスト（fetch）で試みたが、403 Forbiddenエラー — サーバーがボットからのアクセスを拒否
  • 解決策: ブラウザ自動操作（OpenClawのbrowserツール）を使い、実際のブラウザとしてページにアクセス
  • ページのHTML構造を解析し、アブストラクトのテキストを抽出
ここで学ぶこと: HTTPステータスコード（403の意味）、スクレイピングの基本、ブラウザ自動操作
Step 4

AIによる日本語要約

取得した英語のアブストラクトをBlue（AI）が日本語に要約する。単なる翻訳ではなく、Kaiが読んで面白いと思えるポイントを抽出してまとめる。

  • 研究の新規性、手法の特徴、インパクトなど、読者が気になるポイントを重点的に記述
  • タグ付けも自動で行い、分野ごとの検索性を確保
ここで学ぶこと: AIによるテキスト処理、要約と翻訳の違い、ドメイン知識の活用
Step 5

Discord投稿

OpenClawのmessageツールを使ってDiscordの特定チャンネルに投稿する。

  • フォーマット: タイトル（日本語訳）、著者、DOIリンク、要約、タグ
  • TOC画像（論文のグラフィカルアブストラクト）も可能な場合は添付
  • Discordのマークダウンに合わせた整形処理
# 投稿フォーマットの例
message = f"""**{title_ja}**
{authors}

{summary_ja}

{tags}
[DOI]({doi_url})"""
ここで学ぶこと: Discord APIの基本、メッセージフォーマット、画像添付
Step 6

自動実行（cron）

最初はHEARTBEAT（OpenClawの定期チェック機能）で実行していたが、より正確なタイミングのためにcronジョブに移行した。

  • HEARTBEATは会話コンテキスト内で動作し、タイミングはおおよそ
  • cronはOSレベルで正確なスケジュール実行が可能
  • 毎日8:00と20:00（JST）に自動実行
# crontab -e
0 8,20 * * * /path/to/journal-bot/run.sh
ここで学ぶこと: cronとは何か、定期実行の設計、HEARTBEAT vs cronの使い分け

Kaiのレビューで変わったこと

  • 投稿フォーマットの調整 — Kaiが見やすい形にフィードバックし、情報の優先順位を整理
  • HEARTBEATからcronへの移行はKaiの提案（「もっと正確なタイミングで動かしたい」）
  • 対象ジャーナルの拡張 — まずNatureから始めて、今後他のジャーナルも追加可能な設計に

使用技術まとめ

  • Python
  • feedparser
  • ブラウザ自動操作
  • Discord API (messageツール)
  • cron
  • JSON状態管理
