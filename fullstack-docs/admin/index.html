管理画面

認証付きの管理インターフェースで顧客データとステータスを管理

概要

管理者が顧客データの確認・ステータス変更を行うための管理画面を構築した。Basic認証による保護、一覧・詳細表示、ステータス管理機能を実装。

制作ステップ

Step 1

認証の実装

Basic認証で管理画面を保護し、認証されていないアクセスを遮断する仕組みを実装した。

ここで学ぶこと: Basic認証 HTTPヘッダー 認証ミドルウェア
Step 2

顧客一覧と詳細

DBから顧客データを取得し、一覧画面と詳細画面をテンプレートエンジンで表示する機能を実装した。

ここで学ぶこと: テンプレートエンジン(Jinja2) ORM ページネーション
Step 3

ステータス管理

管理者が顧客のステータスを変更できる機能を実装。操作履歴も記録し、誰がいつ何を変更したかを追跡可能にした。

ここで学ぶこと: CRUD操作 ステータスバッジ 操作ログ

ふりかえり

  • Basic認証で最小限のセキュリティを確保しつつ、迅速に管理画面を構築
  • Jinja2テンプレートにより、サーバーサイドレンダリングでシンプルなUI実装
  • 操作ログの導入で、管理操作の透明性と追跡可能性を確保

使用技術

  • Python
  • FastAPI
  • Jinja2
  • SQLite
  • HTML
  • CSS
