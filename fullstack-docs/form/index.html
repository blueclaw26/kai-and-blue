申込フォーム

顧客からのデータ入力を受け付け、DBに保存するフォームシステム

概要

顧客がサービスに申し込むためのWebフォームを設計・実装した。入力項目の最適化、バリデーション、サーバーサイド処理、ステートマシンによる状態管理まで、フォームの全工程を記録。

制作ステップ

Step 1

フォーム設計

入力項目の最適化、セクション分け（基本情報→詳細→口座情報）を行い、ユーザーの離脱を最小限にするフォーム構造を設計した。

ここで学ぶこと: フォームUX 入力バリデーション セクション分け
Step 2

サーバーサイド処理

FastAPIでフォームデータを受け取り、バリデーション後にSQLiteデータベースに保存する処理を実装した。

ここで学ぶこと: FastAPI POSTリクエスト SQLite
Step 3

ステートマシン

申込から完了までの状態遷移を定義し、各状態で許可される操作を制御した。不正な状態遷移を防ぎ、データの整合性を保つ。

ここで学ぶこと: ステートマシン 状態遷移図 トランザクション

ふりかえり

  • セクション分けにより、長いフォームでもユーザーの離脱率を低減できた
  • FastAPIの型ヒントで、サーバーサイドバリデーションを簡潔に実装
  • ステートマシンの導入で、不正な状態遷移をシステムレベルで防止

使用技術

  • Python
  • FastAPI
  • SQLite
  • HTML
  • CSS
