天気ダッシュボード

天気情報ダッシュボードの制作記録

概要

都市名を入力すると現在の天気と7日間予報を表示するダッシュボード。Open-Meteo API（無料、APIキー不要）を使用。

制作ステップ

Step 1

外部APIの調査

Open-Meteo APIの仕様を調査。無料で認証不要、2つのAPIを組み合わせて使う。

  • Geocoding API: 都市名→緯度経度の変換
  • Forecast API: 緯度経度→天気データの取得
ここで学ぶこと: REST API APIドキュメントの読み方 ジオコーディング
Step 2

fetch/async-awaitによるデータ取得

fetch APIでHTTPリクエストを送信し、async/awaitで非同期処理を読みやすく記述する。

  • レスポンスのJSONパースと必要なデータの抽出
  • Geocoding → Forecast の2段階リクエスト
ここで学ぶこと: fetch API async/await JSONパース
Step 3

天気データの表示

APIから取得したデータを画面に表示する。天気コードをUnicode絵文字にマッピングしてアイコンとして使う。

  • 現在の天気: 気温、体感温度、湿度、風速、天気コード→アイコン変換
  • 7日間予報: 日付、最高/最低気温、天気アイコン
ここで学ぶこと: データマッピング DOM操作でUI構築
Step 4

UXの改善

ユーザー体験を向上させるための機能を追加する。エラーが起きても画面が壊れないようにする。

  • ローディング表示（データ取得中のフィードバック）
  • エラーハンドリング（都市が見つからない、API接続失敗）
  • °C/°F切り替え
  • 最後に検索した都市をlocalStorageに保存
  • デフォルトでTokyoを表示
ここで学ぶこと: ローディング状態 エラーハンドリング localStorage
