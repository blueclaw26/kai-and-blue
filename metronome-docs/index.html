メトロノーム

メトロノームアプリの制作記録

概要

BPM制御、タップテンポ、拍子設定、サブディビジョンを備えたメトロノームアプリ。Web Audio APIによる正確なタイミング制御が特徴。

制作ステップ

Step 1

Web Audio APIによる音の生成

ブラウザ内で音を生成するWeb Audio APIを使用。ノイズベースのクリック音を作り、アクセント・通常ビート・サブディビジョンで音色を変える。

  • ホワイトノイズ → ハイパスフィルター → 短いエンベロープでクリック音を生成
  • フィルター周波数と音量でアクセント・通常ビート・サブディビジョンを差別化
ここで学ぶこと: Web Audio API ノイズ生成 BiquadFilter
Step 2

正確なタイミング制御（AudioContext Scheduling）

setIntervalだけではタイミングがずれる（ブラウザのイベントループに依存するため）。audioCtx.currentTimeを使った先読みスケジューリングで正確な再生を実現。

  • 25msごとにスケジューラーが走り、100ms先までの音をあらかじめ予約
  • AudioContextの高精度タイマーにより、音楽用途に耐えるタイミング精度を確保
ここで学ぶこと: AudioContext scheduling 先読みバッファ setIntervalの限界
Step 3

BPM制御とタップテンポ

スライダー、数値入力、±ボタンの3通りの操作方法を実装。タップテンポでは直近4回のタップ間隔の平均からBPMを算出する。

  • 3つの入力手段（スライダー・数値入力・±ボタン）を同期
  • タップテンポ: 直近4回のタップ間隔の平均からBPMを算出
  • 2秒間タップがなければリセット
ここで学ぶこと: タップテンポの算出 複数の入力手段
Step 4

拍子とサブディビジョン

拍子設定（2/4, 3/4, 4/4, 5/4, 6/8, 7/8）に対応。1拍目は自動でアクセント（異なる音色＋視覚的なアンバー色）。サブディビジョンで8分音符、3連符、16分音符の分割が可能。

  • 拍子によってビートインジケーターの数とアクセント位置が変わる
  • サブディビジョン: 1拍を2分割（8分音符）、3分割（3連符）、4分割（16分音符）
ここで学ぶこと: 拍子の概念 サブディビジョン
Step 5

ビジュアルビートインジケーター

拍数分の円を表示し、現在の拍を色で強調。通常拍はティール、アクセントはアンバー。サブディビジョンは小さい点で表示。

  • 拍子が変わるたびにインジケーターの数を動的に変更
  • 音のスケジューリングとUIの更新を同期させる
ここで学ぶこと: 音とUIの同期 動的なUI生成
Step 6

キーボードショートカット

Space: 開始/停止、T: タップテンポ、↑↓: BPM ±1。練習中にマウスを触らずに操作できる。

ここで学ぶこと: keydownイベント

Kaiのレビューで変わったこと

  • 電子音（オシレーター）のピコピコ感を指摘 → ノイズベースのクリック音に変更
  • 正確なテンポが重要なため、AudioContext schedulingで精度を確保
