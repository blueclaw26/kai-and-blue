Kai & Blue

ホームページの制作記録

概要

Kai & Blueの活動を記録するホームページ。HTMLとCSSだけで構成されたシンプルなサイトで、Vercelにデプロイして公開している。

制作ステップ

Step 1

HTML構造の設計

ページの骨格をHTMLで作成。ヘッダー、セクション、フッターのセマンティックな構造を組み立てる。

  • セマンティックHTMLタグ（<header>、<section>、<footer>）で意味のある構造を定義
  • Google Fontsの読み込み（Inter + Noto Sans JP）で日英両対応のタイポグラフィ
ここで学ぶこと: HTMLタグの意味セマンティックHTML外部リソースの読み込み
Step 2

CSSでデザイン

色、フォント、余白、レイアウトをCSSで定義し、ビジュアルを作り込む。

  • CSS Gridで2列・3列のカードレイアウトを構築
  • ホバーエフェクト（translateY、box-shadow）でインタラクティブな操作感
  • レスポンシブ対応 — メディアクエリで画面幅に応じてカラム数を変更
ここで学ぶこと: CSSプロパティGridレイアウトメディアクエリトランジション
Step 3

Vercelへのデプロイ

GitHubリポジトリにコードをpushし、Vercelが自動でデプロイする仕組みを構築。

  • GitHubリポジトリにコードをpush
  • VercelがGitHubと連携して自動デプロイ
  • pushするたびに自動で最新版が公開される
ここで学ぶこと: Git/GitHubの基本CI/CDの概念ホスティングサービス
Step 4

デザインの反復改善

Kaiのフィードバックを受けて、デザインと構造を何度も改善した。

  • 最初のダークブルーテーマから、ウォームグレーのライトテーマに変更
  • 番号リストから3列グリッドに構造を変更
  • 56プロジェクトのロードマップを4フェーズで構成
  • ホバーツールチップを追加（CSS方式→JS方式に修正）
ここで学ぶこと: デザインの反復プロセスフィードバックの反映CSS vs JSの使い分け

Kaiのレビューで変わったこと

  • ダークブルー→ライトテーマへの方向転換
  • 「AIっぽくない」デザインの追求
  • 番号リストの廃止（「ダサい」というフィードバック）
  • Phaseの説明文を削除（「幼稚に見える」）
  • ツールチップのclipping問題の報告→JS方式に修正

使用技術まとめ

  • HTML
  • CSS
  • CSS Grid
  • Vercel
