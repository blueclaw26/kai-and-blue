日次レポート自動化

CRMデータを毎日集計してGoogle Sheetsに書き込む仕組みの制作記録

概要

CRMから取引データを毎日自動で取得し、カテゴリ別の件数と金額を集計して、Google Sheetsの指定セルに書き込む自動化ツール。

制作ステップ

Step 1

CRM APIの調査

CRMのデータにAPIでアクセスする方法を調査し、最適なAPIバージョンを選定した。

  • CRMのデータにAPIでアクセスする方法を調査
  • v1 APIとv2 APIの違いを検証（v1のフィルタリングが不正確であることを発見）
  • v2を使うことに決定 — より正確なフィルタリングとアーカイブ済みデータの自動除外
ここで学ぶこと: REST APIの基本、APIバージョニング、エンドポイントの調査方法
Step 2

データの取得と分類

APIから取得した取引データを、ビジネスロジックに基づいて分類する。

  • v2 APIから取引データを取得
  • カスタムフィールドの構造を解析（CRMは独自のフィールドを持っていることが多い）
  • 取引の種別に応じた分類ロジックを実装
ここで学ぶこと: APIレスポンスの構造理解、カスタムフィールドの扱い、データ分類ロジック
Step 3

金額の集計

分類された取引ごとに金額を合算し、日次の集計データを生成する。

  • カテゴリ別に金額を合算
  • 日付フィルタリング（当日分のみ抽出）
  • CRMのフィルター機能（filter_id）も併用してサーバー側で絞り込み
ここで学ぶこと: データの集計処理、日付フィルタリング、クライアント側 vs サーバー側フィルタリング
Step 4

Google Sheets連携

集計結果をGoogle Sheetsの特定セルに自動で書き込む仕組みを構築した。

  • Google Cloud Platformでサービスアカウントを作成
  • サービスアカウントにスプレッドシートへの編集権限を付与
  • googleapis ライブラリでシートの特定セルに書き込み
  • 日付文字列で行を検索して特定し、指定列に値を書き込み
ここで学ぶこと: サービスアカウント認証、Google Sheets API、セル指定の仕組み
Step 5

スクリプト化

一連の処理を再利用可能なスクリプトとしてまとめた。

  • 一連の処理をNode.jsスクリプトにまとめる
  • 環境変数やAPIキーの管理（.gitignoreでキーファイルを除外）
  • エラーハンドリング（API接続失敗時のリトライ等）
ここで学ぶこと: スクリプトの構造化、環境変数管理、.gitignoreの役割
Step 6

cron自動実行

毎日決まった時刻に自動で集計・書き込みを実行し、結果をDiscordに報告する。

  • 毎日14:00（中間集計）と19:30（最終集計）にcronで自動実行
  • 実行結果をDiscordに自動報告
ここで学ぶこと: cronスケジューリング、複数回実行の設計、実行結果の通知

Kaiのレビューで変わったこと

  • v1 APIの不正確さをKaiが実データで確認し、v2への移行を決定
  • 書き込み先のシート構造（列の割り当て）はKaiが指定
  • 「コード書いたら毎回GitHubにpush」ルールの確立（このプロジェクトがきっかけ）
  • 14:00の中間集計追加はKaiの要望（「夕方まで待たずに途中経過が見たい」）

使用技術まとめ

  • Node.js
  • CRM API v2
  • Google Sheets API
  • サービスアカウント認証
  • cron
  • Discord通知
