MiroFish

AI同士が対話する将来シミュレーションツールのセットアップ記録

概要

MiroFishは複数のAIエージェントがSNS上の社会をシミュレーションし、未来を推演するオープンソースのAI予測エンジン。既存のOSSプロジェクトを自分の環境にセットアップして動かす経験として取り組んだ。

制作ステップ

Step 1

プロジェクトの調査と選定

  • GitHubでMiroFishのリポジトリを調査
  • 技術スタック（Next.js + Flask/Python + OASISエンジン）を確認
  • ナレッジグラフ（GraphRAG）によるAIエージェント間の関係性モデリングに注目
ここで学ぶこと: OSS 技術スタック ナレッジグラフ
Step 2

開発環境の構築

  • Python 3.12のインストール（3.14では一部ライブラリがビルドできない）
  • uvパッケージマネージャのインストール（pipより高速な依存解決）
  • Rustコンパイラのインストール（tiktokenのビルドに必要）
ここで学ぶこと: Pythonバージョン管理 パッケージマネージャ ネイティブ拡張のビルド
Step 3

依存パッケージのインストール

  • フロントエンド: npm run setupでNext.jsの依存をインストール
  • バックエンド: uv sync --python python3.12でPython依存をインストール
  • 依存解決で発生するエラーへの対処（バージョン競合、ビルド失敗等）
ここで学ぶこと: npm 依存解決 ビルドエラーの対処
Step 4

環境変数の設定

  • .envファイルの作成（テンプレートからコピー）
  • LLM APIキー（OpenAI互換）の設定
  • Zep Cloud APIキー（ナレッジグラフ用）の設定
  • API互換性の概念（OpenAI互換APIなら異なるプロバイダーでも動く）
ここで学ぶこと: 環境変数(.env) API互換性
Step 5

フロントエンドとバックエンドの起動

  • npm run devで全体を起動（フロント:3000 + バックエンド:5001）
  • ポートの概念と複数サービスの同時実行
ここで学ぶこと: ポート フロント/バックエンド分離
Step 6

シミュレーションの実行

  • 複数AIエージェントがSNS上で発言・議論・予測を行う
  • トークン消費が大きいため、最初は40ラウンド以下で実行
  • シミュレーション結果のナレッジグラフでの可視化
ここで学ぶこと: AIエージェント トークン消費

Kaiのレビューで変わったこと

  • KaiがMiroFishを見つけてきてBlueに共有（Kaiが主導したプロジェクト選定）
  • ナレッジグラフへの興味がプロジェクト選定の決め手
  • セットアップ中のPythonバージョン問題（3.14→3.12に変更）はBlueが検証して解決

使用技術まとめ

  • Python
  • Node.js
  • JavaScript
